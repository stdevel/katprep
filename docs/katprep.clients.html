Validates that the Foreman API uses a FQDN as hostname. Also looks up the “real” hostname if “localhost” is specified. Otherwise, the picky Foreman API won’t connect.

a valid HTTP method and a sub-URL (such as /cgi-bin/status.cgi). Optionally, you can also specify payload (for POST). There are also alias functions available.
